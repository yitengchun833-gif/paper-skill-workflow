# AGENTS.md

## 项目定位

本仓库是一个论文写作与论文处理工作流 Skill 集合，用于辅助完成：

- 查文献与阅读文献
- 论文大纲与章节写作
- 图表、数据和科研输出处理
- Word 论文格式识别与分模块修改
- 学术表达优化与重复率风险控制
- AIGC 风格风险报告分析
- 多角色审查与最终集成

默认目标是提高论文质量、格式合规性、证据一致性和表达自然度，而不是规避学校诚信要求或伪造研究结果。

---

## 总原则

1. 不编造文献、DOI、作者、年份、期刊、数据、图表、实验过程、仿真参数或结论。
2. 没有原始数据时，只能写“由图读数估算”“近似读数”“从图中趋势看”等限定表达。
3. 不把文献结论写成本论文实验或仿真结果。
4. 不把“焦耳热功率密度”直接写成“晶格温度”。
5. 不把“热源分布变化”直接写成“温度一定降低”。
6. 不承诺 AIGC、查重或检测结果一定降低到某个百分比。
7. 不为绕过检测而改写文本，只做学术表达自然化、证据绑定和重复风险控制。
8. Word 格式任务默认只改格式，不改正文内容。
9. 修改正式论文文件前，必须先创建副本或输出新文件，不覆盖原文件。
10. 每次完成修改后，必须输出修改说明、风险说明和未修改区域说明。

---

## Skill 调用总顺序

遇到论文任务时，先判断任务类型，再选择对应 Skill。不要一次性调用全部 Skill。

推荐顺序：

1. 材料识别与任务判断
2. 查文献或阅读文献
3. 写大纲或章节
4. 图表与数据分析
5. 学术表达优化
6. Word 格式处理
7. 最终审查
8. 输出修改报告

如果用户只要求单一任务，例如“只套格式”或“只分析检测报告”，只调用相关 Skill，不启动完整流程。

## Skill 路由规则

### 1. AIGC 报告驱动的表达优化

使用 Skill：

- `aigc-academic-expression-optimizer`
- 必要时配合 `aigc-academic-expression-optimization`
- 生成 Word 时配合 `documents:documents`

触发场景：

- 用户上传 AIGC 检测报告
- 用户要求根据报告优化高风险段落
- 用户要求继续降低 AI 风格风险
- 用户要求按检测报告定位修改

默认流程：

1. 先读取检测报告。
2. 提取总体疑似度、章节风险和高/中/轻风险片段。
3. 在 Word 中定位对应段落。
4. 只处理报告标出的风险段落。
5. 高风险大改，中风险中改，轻风险轻改。
6. 保留数据、图号、公式编号、表号、参考文献编号和格式。
7. 输出新版 Word 和修改记录。

禁止事项：

- 不全文无差别大改。
- 不承诺复检一定达到某个数值。
- 不伪造作者经历、实验过程、仿真过程或文献。
- 不删除图表、公式、参考文献或页眉页脚。

---

### 2. 段落级学术表达优化

使用 Skill：

- `aigc-academic-expression-optimization`

触发场景：

- 用户只提供一段或几段论文文本
- 用户要求降低模板化表达
- 用户要求让段落更像本人基于图表和仿真写出来的分析

默认处理方式：

1. 先识别原段落中的事实、图表、变量、数据和结论边界。
2. 删除空泛套话。
3. 增加图号、偏置条件、读数来源、限制性判断。
4. 保留专业术语和原意。
5. 输出“修改后文本”和“修改说明”。

---

### 3. Word 论文格式处理

使用 Skill：

- `thesis-format-module-editor`
- 生成或编辑 DOCX 时配合 `documents:documents`

触发场景：

- 用户要求套学校格式
- 用户要求模仿参考论文格式
- 用户要求修改目录、标题、页码、页眉页脚、图表编号或公式格式
- 用户明确要求“不改正文，只改格式”

默认流程：

1. 先确认材料是否齐全：
   - 当前论文 Word
   - 学校撰写规范
   - 参考论文 Word
   - 用户截图或具体要求
2. 先创建副本。
3. 按模块处理：
   - M0 范围锁定与备份
   - M1 页面设置、节、页眉页脚、页码
   - M2 标题层级与编号
   - M3 正文段落格式
   - M4 目录格式与跳转
   - M5 图题、表题、图表编号
   - M6 公式格式与编号
   - M7 参考文献与附录格式
   - M8 最终渲染检查
4. 每完成一个模块，输出修改说明。
5. 不确定时询问用户，不擅自推断。

禁止事项：

- 不把自动目录改成纯文本目录。
- 不随意重排图表编号或公式编号。
- 不修改正文语义。
- 不替换参考文献条目。
- 不覆盖原文件。
- 不一次性全改后才汇报。

### 4. 论文写作与结构重构

使用 Skill：

- `nature-writing`

触发场景：

- 用户要求写摘要、引言、结果、讨论、结论或章节初稿
- 用户要求根据图表、数据、实验过程或仿真条件写论文内容
- 用户要求重构论文逻辑

默认流程：

1. 先确认论文题目、章节、研究对象、核心变量和已有证据。
2. 明确哪些内容来自用户材料，哪些内容来自文献。
3. 只基于已提供材料写作。
4. 缺少证据时写成占位或提示用户补充。
5. 输出后注明该段依赖的图、表、数据或文献来源。

禁止事项：

- 不凭空写实验结果。
- 不为凑字数加入空泛意义句。
- 不把未验证结论写成确定结论。
- 不新增用户未提供的参数。

---

### 5. 学术英文润色与翻译

使用 Skill：

- `nature-polishing`

触发场景：

- 用户要求润色英文摘要
- 用户要求中译英或英文学术表达优化
- 用户要求提高论文英文段落质量

默认流程：

1. 先判断是翻译、润色还是结构重写。
2. 保留术语、变量、单位、图号和参考文献。
3. 优先保证准确，再提升表达。
4. 输出时可附简短修改说明。

禁止事项：

- 不把中文原意改成更强结论。
- 不用过度夸张的 Nature 风格表达。
- 不引入新实验、新数据或新引用。

### 6. 审稿意见回复

使用 Skill：

- `nature-response`

触发场景：

- 用户提供导师意见、评阅意见、审稿意见或修改建议
- 用户要求写逐条回复、答辩回应或修回说明

默认流程：

1. 先拆分每条意见。
2. 判断意见类型：
   - 格式问题
   - 数据问题
   - 逻辑问题
   - 语言问题
   - 文献问题
3. 给出修改措施。
4. 写成礼貌、具体、可核查的回复。
5. 不承诺没有实际完成的修改。

### 7. 文献检索

使用 Skill：

- `nature-academic-search`

触发场景：

- 用户要求查文献
- 用户要求找 DOI、BibTeX、RIS、PMID 或 arXiv
- 用户要求验证文献真实性

默认流程：

1. 先从论文题目、关键词和研究问题生成检索式。
2. 优先使用可信数据库或正式出版信息。
3. 记录标题、作者、年份、期刊、DOI 和用途。
4. 区分核心文献、背景文献和可选文献。
5. 不确定的文献必须标注“待核验”。

禁止事项：

- 不编 DOI。
- 不编期刊卷期页码。
- 不把搜索结果当成已经阅读过的全文。
- 不只凭标题判断文献结论。

### 8. 正文补引用

使用 Skill：

- `nature-citation`

触发场景：

- 用户要求给论文正文补参考文献
- 用户要求句段到参考文献映射
- 用户要求导出引用文件

默认流程：

1. 先拆分需要引用的句段。
2. 判断每句话需要什么类型证据。
3. 为每个关键论断匹配文献。
4. 标注引用支撑范围。
5. 输出参考文献列表或引用文件。

禁止事项：

- 不给没有证据的句子硬塞引用。
- 不把不相关文献用于支撑关键结论。
- 不把二手综述当成所有原始数据来源。

### 10. 数据可用性与数据声明

使用 Skill：

- `nature-data`

触发场景：

- 用户要求写数据可用性声明
- 用户要求选择数据仓库
- 用户要求准备 FAIR 元数据

默认流程：

1. 先确认数据类型、是否公开、是否敏感。

### 11. 科研图件处理

使用 Skill：

- `nature-figure`

触发场景：

- 用户要求制作、重绘、审查或导出科研图
- 用户要求生成多面板图、曲线图、热图或论文插图

默认流程：

1. 先确认图的结论。

4. 生成图前明确坐标、单位、图例和导出格式。

禁止事项：

- `nature-paper2ppt`

- 用户要求从论文生成答辩 PPT、组会 PPT、汇报 PPT

### 13. 多角色审查

使用 Skill：

- `multi-agent-workflow`

触发场景：

- 用户要求多角度审查论文
- 用户要求从逻辑、格式、数据、语言、文献等维度分别检查
- 用户要求复杂任务拆分执行

默认流程：

- 数据与图表审查员

## 文件保护规则

正式论文文件必须遵守：

1. 原文件只读。
2. 所有修改输出到新文件。
3. 新文件名必须包含任务类型和日期。
4. 修改前后必须有记录。
5. 如果工具无法完整保留格式，必须停止并提示用户。
6. 不得删除封面、诚信承诺书、目录、参考文献、致谢、图表和公式。

建议文件命名：

```text
原文件名_格式修订版_YYYYMMDD.docx
原文件名_表达优化版_YYYYMMDD.docx
原文件名_最终检查版_YYYYMMDD.docx
```

- 至少一张论文图或仿真图

测试指令：

要求：
1. 先识别当前文件夹中有哪些材料。
2. 判断应该调用哪些 Skill。

## 推荐实战流程

### 流程 A：AIGC 报告优化

1. 上传当前最新版论文。
2. 上传最新检测报告。
3. 调用 `aigc-academic-expression-optimizer`。
4. 生成风险清单。
5. 定位 Word 段落。
6. 分级修改。
7. 输出新版 Word 和修改记录。
8. 用户复检。
9. 根据新报告继续局部优化。

### 流程 B：论文套格式

1. 上传当前论文 Word。
2. 上传学校格式规范。
3. 上传参考论文 Word 或截图。
4. 调用 `thesis-format-module-editor`。
5. 先处理标题与目录。
6. 再处理页码、页眉页脚。
7. 最后处理图表、公式和参考文献。
8. 输出格式修订版和问题清单。

### 流程 C：查文献并补引用

1. 用户提供论文题目和章节。
2. 调用 `nature-academic-search`。
3. 查询并核验文献。
4. 调用 `nature-citation`。
5. 建立正文句段与参考文献对应关系。
6. 输出参考文献列表和引用建议。

### 流程 D：论文最终审查

1. 调用 `multi-agent-workflow`。
2. 按角色检查：
   - 逻辑
   - 数据
   - 图表
   - 格式
   - 引用
   - 语言
3. 主线程整合问题。
4. 输出最终修改清单。
5. 不直接改正文，除非用户明确要求。

## 针对理工类仿真论文的特别规则

如果论文涉及 AlGaN/GaN HEMT、TCAD、Silvaco、Atlas、焦耳热、自热效应、温度云图或电热耦合，必须遵守：

1. 区分底部衬底材料变化和其他结构参数变化。
2. 如果只有焦耳热图，不写晶格温度定量结论。
3. 如果只有图像读数，不写成原始导出数据。
4. 如果比较蓝宝石和金刚石衬底，重点应放在热导率差异对热源分布和电热耦合趋势的影响。
5. 不随意新增 VGS、VDS、功率、温度、掺杂浓度或材料参数。
6. 金刚石衬底优势可以从热导率和热扩散角度解释，但论文结论必须受限于已有仿真图和曲线。
7. 图像分析时必须说明图像变量、偏置条件、色标含义和读数边界。

推荐表达：

- “在当前仿真条件下”
- “从图中趋势看”
- “由图读数估算”
- “该结果更适合解释为”
- “不能直接等同于”
- “仍需结合晶格温度场进一步验证”

避免表达：

- “充分证明”
- “显著降低温度”
- “完全解决自热问题”
- “具有广阔应用前景”
- “为后续研究奠定坚实基础”

## 交互规则

1. 用户要求一步一步来时，不一次性给出长流程。
2. 用户只问下一步时，只回答下一步。
3. 用户上传文件后，先识别文件类型，再决定 Skill。
4. 用户要求修改文件时，必须说明是否会改原文件。
5. 用户明确限制修改范围时，严格遵守。
6. 如果材料缺失，先指出缺失材料，不猜。
7. 如果结论不确定，明确写“不确定”或“材料不足以判断”。
8. 不输出冗长客套话。

## 失败处理

如果无法读取报告：

```text
当前报告无法准确提取风险片段。请上传清晰版 PDF 或包含标色正文页面的截图。
```

如果无法定位 Word 段落：

```text
报告中的风险片段在 Word 中无法定位，可能是检测报告对应的不是当前 Word 版本。请确认上传的是本次检测使用的论文版本。
```

如果格式修改会破坏原文件：

```text
当前操作可能破坏原有目录、页码或交叉引用。建议先输出副本并做局部测试，不直接覆盖原文档。
```

如果用户要求伪造数据或文献：

```text
不能伪造数据、文献或实验过程。可以改为基于已有材料重新组织表达，并标注当前证据边界。
```

## 完成标准

一次任务完成时，必须满足：

- 已说明调用了哪些 Skill。
- 已说明处理了哪些文件。
- 已说明改了什么、没改什么。
- 已保护原文件。
- 已输出结果文件或可复制文本。
- 已说明不确定点和下一步建议。
